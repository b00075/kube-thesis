Introducing Kubernetes

Development is moving away from monolithic applications where a delay in a single element could mean missing a release deadline and is being replaced with developing a series of microservices, where each element can be developed in isolation removing roadblocks to releases. To facilitate this model of development a containerized solution is necessary. In the past monolithic apps were deployed to a host, using the hosts operating system's packet manager which tied it to the underlying hardware resulting in heavy and non portable solutions. With containerization the app and its necessary libraries and dependencies are bundled into a lightweight container and are deployed to a host using operating system virtualization so all containers are isolated from each other, the host machine's file system and hardware.
From Kubernetes Docs

  • Kubernetes is the defacto platform for container orchestration.
  • Kubernetes is an open-sourced project released in 2014 and written in Go.
  • It is a portable platform for managing containerized workloads and services.
  • Kubernetes orchestrates computing, networking, and storage infrastructure.
  • Kubernetes has been designed as a platform that can be configured and extended to suit the user/organization needs.
  • The same API is available to developers and users to create their own components such as schedulers, controllers etc.
  • Kubernetes can be thought of as a microservices platform, a portable cloud platform and a container platform among others.
  • Kubernetes is not a Continuous Delivery solution so you cannot deploy source code and expect Kubernetes to build and deploy it.
  • Kubernetes does not have a storage solution as a built-in service, but can be extended to include storage plugins.
