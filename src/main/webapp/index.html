Pre-requisites

  • Download the three virtual machines and start them up (assumes you have Oracle Virtual Box installed) The Kubernetes cluster is running on the host only network adapter set to the 172.42.42.1/24 network. If the vm doesn't start check that you have this set.
  • Root user/password = root:kubeadmin
  • Standard user/password = vagrant:vagrant
    • kmaster - md5 checksum f29c29eb6883a916c3ed58fe1f267e12
    • kworker1 - md5 checksum 831808a090ef393330e386bb4e0a1ab1
    • kworker2 - md5 checksum fca6f53938857f020fbfc9590c69b8a4
  • Use curl to download the correct Kubernetes command line tool for your OS
    • Windows - curl -LO https://storage.googleapis.com/kubernetes-release/release/v1.9.0/bin/windows/amd64/kubectl.exe
      • Add the binary to your PATH
    • MacOS - curl -LO https://storage.googleapis.com/kubernetes-release/release/v1.9.0/bin/darwin/amd64/kubectl
      • Make the binary executable - chmod +x /kubectl
      • Move the binary to your PATH - mv /kubectl /usr/local/bin/kubectl
    • Linux - curl -LO https://storage.googleapis.com/kubernetes-release/release/v1.9.0/bin/linux/amd64/kubectl
      • Make the binary executable - chmod +x /kubectl
      • Move the binary to your PATH - mv /kubectl /usr/local/bin/kubectl
  • Navigate to your home directory and create a new hidden folder .kube and create config file- cd ~/ && mkdir .kube && cd .kube && touch config
    Or on Windows:
    mkdir .kube
    type nul > config
  • ssh into kmaster and copy kubernetes config file
    • ssh root@172.42.42.100 (password is kubeadmin)
    • display the config file - cat /etc/kubernetes/admin.conf
    • copy contents and paste into config file created in the previous step on your local machine
    • To ensure all is working open a terminal/command prompt and run kubectl cluster-info
    • You should recieve a response similair to the following:
      Kubernetes master is running at https://172.42.42.100:6443
      KubeDNS is running at https://172.42.42.100:6443/api/v1/namespaces/kube-system/services/kube-dns:dns/proxy
